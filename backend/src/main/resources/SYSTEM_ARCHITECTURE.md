# Video-Platform 系统架构说明

## 1. 项目整体定位

本项目是一个面向视频内容分发场景的全栈系统，核心目标是实现视频上传、播放、互动、搜索、推荐、私信与通知等能力。整体采用前后端分离架构，后端为 **纯 Java** 实现，不依赖独立 Python agent 服务。

系统当前由两个主要模块组成：

- `backend`：Spring Boot 后端，负责业务逻辑、数据访问、鉴权、缓存、对象存储、搜索、推荐、消息队列等
- `frontend`：Vue 3 前端，负责页面展示、用户交互和接口调用

## 2. 总体架构

```text
用户浏览器
   |
   v
Vue3 前端
   |
   v
Spring Boot 后端
   |
   |-- MySQL：业务主数据
   |-- Redis：缓存、状态数据、热点数据
   |-- Elasticsearch：搜索索引
   |-- MinIO：视频、封面、头像、消息图片等对象存储
   |-- RocketMQ：异步任务、解耦处理
```

## 3. 后端核心分层

- `controller`：对外暴露 REST 接口，处理参数接收与结果返回
- `service`：承载业务逻辑，协调数据库、缓存、搜索、消息队列等基础设施
- `mapper`：基于 MyBatis-Plus 的数据访问层
- `entity`：数据库实体对象
- `model/vo`：接口返回对象，用于页面展示数据封装
- `config`：Spring、Redis、跨域、拦截器等配置
- `mq`：异步消息生产与消费逻辑
- `utils`：JWT、上下文、工具类等通用能力

## 4. 主要业务域

### 4.1 用户域
- 用户注册与登录
- JWT 鉴权
- 用户主页与创作者信息
- 头像上传与访问

### 4.2 视频域
- 视频上传、删除、详情、播放统计、观看历史
- `VideoServiceImpl` 作为门面层
- `VideoQueryService` 负责查询类逻辑
- `VideoCommandService` 负责上传、删除、播放记录等写操作
- `VideoViewAssembler` 负责聚合并组装展示对象

### 4.3 互动域
- 点赞、收藏、评论、弹幕、关注等互动能力
- 列表聚合阶段批量加载互动状态与统计数据

### 4.4 消息域
- 私信会话
- 通知与系统消息
- 消息中心汇总与已读状态维护

### 4.5 搜索域
- Elasticsearch 关键词搜索
- 用户搜索、热门搜索、搜索历史
- `/search/hybrid` 接口保留，当前实现为 ES-only

### 4.6 内容补全与规则增强域
- 基于后端本地规则的标签推荐
- 投稿时标题、标签、分类的自动补全
- 基于行为和规则维护视频标签特征与推荐画像

## 5. 核心数据流

### 5.1 视频上传链路

```text
前端上传视频/封面
   ->
后端接收文件
   ->
上传到 MinIO
   ->
写入 MySQL 视频记录
   ->
发送 MQ 消息
   ->
异步处理封面/视频相关后续任务
   ->
失效缓存并刷新查询结果
```

### 5.2 视频列表查询链路

```text
前端请求视频列表
   ->
后端查询视频主列表
   ->
批量加载作者、评论数、点赞状态、收藏状态、观看进度、统计值
   ->
VideoViewAssembler 组装 VideoVO
   ->
返回前端展示
```

### 5.3 投稿补全链路

```text
前端提交轻投稿
   ->
后端检查 title / description / category / tags 是否缺失
   ->
LocalContentAnalysisService 根据文本与本地元数据做规则匹配
   ->
补全标题、标签、分类
   ->
写入 MySQL 与推荐特征表
```
