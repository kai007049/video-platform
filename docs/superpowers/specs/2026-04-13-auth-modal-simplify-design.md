# 登录注册弹窗精简设计

## 1. 背景

用户希望对当前前端登录/注册弹窗做一次轻量精简：

1. 登录时输入项从“邮箱地址”改成“用户名”。
2. 登录弹窗去掉“第三方登录”区域。
3. 注册时只保留“用户名 + 密码”。
4. 注册弹窗也去掉“第三方登录”区域。

该需求目前只涉及前端交互与表单展示层，不要求重构后端登录/注册协议。

## 2. 目标

本次改动目标如下：

1. 登录弹窗视觉上改为“用户名 + 密码”。
2. 登录提交仍复用现有 `account` 字段，不改后端接口。
3. 注册弹窗只保留用户名、密码和现有条款勾选。
4. 登录与注册弹窗都移除“第三方登录”模块。
5. 保持现有切换逻辑、提交流程和成功/失败提示不变。

## 3. 非目标

本次不做以下事情：

1. 不修改后端 `AccountLoginDTO`。
2. 不修改后端 `UserRegisterDTO`。
3. 不增加新的登录方式。
4. 不删除“服务条款”勾选。
5. 不重做弹窗视觉风格，只做字段与内容精简。

## 4. 方案选择

采用 **方案 A：最小改动方案**。

### 4.1 登录弹窗

- 输入框占位文案从“邮箱地址”改为“用户名”。
- 输入框类型从 `email` 改为 `text`，避免浏览器邮箱格式校验干扰。
- 绑定字段仍然使用 `form.account`，这样不需要改动 store 和后端接口。
- 保留密码输入框、忘记密码文案、登录提交按钮。
- 删除“第三方登录”整块 DOM。

### 4.2 注册弹窗

- 保留用户名输入框。
- 删除邮箱输入框。
- 保留密码输入框。
- 保留“我已阅读并同意服务条款”。
- 删除“第三方登录”整块 DOM。
- `form` 数据结构去掉 `email` 字段，避免前端仍保留无用状态。

## 5. 涉及文件

### 修改
- `frontend/src/components/LoginModal.vue`
- `frontend/src/components/RegisterModal.vue`

### 不修改但需保持兼容
- `frontend/src/stores/user.js`
- `frontend/src/api/user.js`
- `backend/src/main/java/com/bilibili/video/model/dto/AccountLoginDTO.java`
- `backend/src/main/java/com/bilibili/video/model/dto/UserRegisterDTO.java`

## 6. 实施细节

### 6.1 LoginModal.vue

需要改动：

1. 将 `type="email"` 改为 `type="text"`。
2. 将 placeholder 从“邮箱地址”改为“用户名”。
3. 可将输入图标从 ✉ 调整为更贴近用户名语义的图标（如 👤），但不是硬要求。
4. 删除 `third-party-login` 区块。
5. 保持 `form.account`、`form.password`、`handleSubmit()` 逻辑不变。

### 6.2 RegisterModal.vue

需要改动：

1. 删除邮箱输入框对应的整段模板。
2. `form` 从 `{ username: '', email: '', password: '' }` 改为 `{ username: '', password: '' }`。
3. 打开弹窗时不再重置 `form.email`。
4. 删除“第三方登录”区块。
5. 保持用户名、密码、条款勾选和注册成功后跳转登录的逻辑不变。

## 7. 验收标准

满足以下条件即可认为改动完成：

1. 登录弹窗只显示“用户名”和“密码”两个输入项。
2. 登录弹窗不再显示“第三方登录”。
3. 注册弹窗只显示“用户名”和“密码”两个输入项（外加条款勾选）。
4. 注册弹窗不再显示“第三方登录”。
5. 登录提交流程仍可正常工作。
6. 注册提交流程仍可正常工作。
7. 前端不因删除 `email` 字段而报错。

## 8. 风险与控制

### 风险

1. 登录框虽然显示“用户名”，但后端实际仍接受 `account` 字段，若前端误改字段名会导致登录失败。
2. 注册框删除邮箱后，如果还有残留重置逻辑或校验逻辑，可能出现运行时错误。

### 控制措施

1. 明确只改展示文案，不改登录提交字段名。
2. 删除邮箱字段时同步清理 `form` 初始化和 reset 逻辑。
3. 改完后至少做一次前端构建或页面手测，验证登录/注册弹窗能正常打开与提交。

## 9. 结论

本次属于纯前端表单精简需求，最佳做法是保持后端协议不变，只调整前端字段展示与无关模块。这样既满足产品要求，也能把改动控制在最小范围。
