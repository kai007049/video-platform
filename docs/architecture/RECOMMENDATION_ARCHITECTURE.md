# 推荐系统架构说明

## 1. 设计目标

推荐系统的目标不是简单返回热门视频，而是在可解释、可调试的前提下，根据用户行为、内容标签、分类偏好、作者偏好和视频热度生成更符合用户兴趣的视频流。

当前版本采用 backend-only 实现，不依赖独立推荐服务。这样做的好处是实现成本可控、链路清晰、方便在个人项目中完整展示推荐系统的核心结构。

## 2. 推荐链路

```text
请求推荐页
   ->
读取用户行为与内容特征
   ->
多路召回候选视频
   ->
构建候选特征
   ->
线性打分排序
   ->
页面级重排
   ->
记录曝光日志
   ->
返回推荐结果
```

## 3. 多路召回

当前召回通道包括：

- 热门召回：从 Redis 热榜中召回高热视频
- 新鲜召回：按发布时间召回近期内容
- 标签兴趣召回：根据用户兴趣标签与视频标签特征匹配
- 分类偏好召回：根据用户观看、点赞、收藏过的分类召回
- 作者偏好召回：根据关注作者和高频互动作者召回
- 运营推荐召回：召回后台标记为推荐位的视频

这样拆分的原因是不同通道解决的问题不同：

- 热门召回保证基础质量
- 新鲜召回保证新内容曝光
- 标签和分类召回保证兴趣相关性
- 作者召回保证关系链和熟悉内容
- 运营召回提供人工干预入口

## 4. 打分模型

当前版本采用可解释的线性打分：

```text
finalScore =
  hotWeight * hotScore
+ freshWeight * freshScore * freshnessDecay
+ tagWeight * interestTagScore
+ categoryWeight * categoryAffinityScore
+ authorWeight * authorAffinityScore
+ editorialWeight * editorialBoost
- recentWatchPenalty
```

这种方式不是复杂推荐模型的最终形态，但非常适合作为推荐系统 V1：

- 便于调权
- 便于排查推荐原因
- 便于向面试官解释系统如何工作
- 便于和 seed 数据验证结果对应起来

## 5. 页面级重排

打分排序后会继续做页面级重排，主要目标是提升 feed 流观感：

- 限制单页同作者数量
- 限制单页同分类数量
- 限制同分类连续出现
- 降低最近看过内容的重复曝光

重排层关注的是“页面体验”，而不是单个视频的相关性分数。

## 6. 冷启动策略

### 新用户冷启动

新用户缺少历史行为，主要依赖：

- 热门召回
- 新鲜召回
- 运营推荐召回

### 新视频冷启动

新视频缺少互动数据，主要依赖：

- 新鲜召回
- 标签兴趣召回
- 分类偏好召回
- 运营推荐召回

这能避免新视频完全被热门内容挤出曝光池。

## 7. 曝光日志

推荐结果会记录曝光日志，包含：

- `videoId`
- `score`
- `channels`
- `strategyVersion`

曝光日志的价值是让推荐系统具备可分析性：

- 可以解释某个视频为什么被推荐
- 可以观察不同召回通道的贡献
- 可以为后续 CTR、完播率、收藏率等指标统计打基础

## 8. 与缓存体系的关系

推荐接口最终返回的是完整视频展示对象，但推荐系统本身更适合处理 `videoId` 候选集。

当前设计上将推荐链路和视频详情缓存解耦：

- 推荐系统负责生成候选视频 ID 和排序
- 视频详情缓存负责承接具体视频展示数据
- 后续推荐结果缓存可以只缓存短时间稳定的 `videoIds`

这样可以避免把完整 `VideoVO` 缓存在推荐层，降低缓存失效复杂度。

## 9. 验证方式

推荐系统可以通过 seed generator 生成演示数据进行验证：

- 生成用户、作者、视频和行为数据
- 固定随机种子，保证结果可复现
- 对比冷启动用户和有行为用户的推荐结果
- 观察热门召回、标签召回、分类召回等通道的差异

这让推荐系统不只是“接口能返回数据”，而是可以通过数据分布和用户画像验证推荐倾向。

## 10. 当前边界

当前推荐系统更适合定义为可解释推荐 V1：

- 规则和线性打分为主
- 适合中小规模数据和演示场景
- 尚未引入复杂机器学习模型
- 尚未形成完整的线上指标闭环

后续可以继续演进：

- 引入点击、完播、跳出、停留时长等反馈信号
- 做推荐结果缓存和热点 key 保护
- 增加召回通道效果统计
- 引入离线特征任务
- 构建 CTR、完播率、收藏率等效果指标

## 11. 面试讲解建议

可以从四个层次讲：

1. 为什么不能只用热门榜
2. 为什么拆成召回、打分、重排
3. 为什么当前阶段选择可解释线性模型
4. 后续如何演进到反馈闭环和更复杂模型

## 12. 总结

当前推荐系统的核心价值是：用可解释、可调试、可验证的方式，把视频平台推荐链路中的召回、排序、重排和曝光分析完整串起来，为后续更复杂的推荐策略留下清晰扩展空间。
